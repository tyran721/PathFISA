# PathFISA 需求分析

## 1. 文档概述

### 1.1 项目名称

PathFISA：病理小样本增量自学习智能标注软件。

### 1.2 产品定位

PathFISA 是一个基于浏览器的病理图像标注与模型迭代平台。它以全切片图像（Whole Slide Image，WSI）和局部病理图像为主要对象，通过人工标注、AI 辅助、医生复核、主动学习选样和受控增量训练，逐步提高特定病种、组织类型或标注任务上的辅助标注能力。

系统首版用于科研、数据生产和算法验证，不直接承担临床诊断，不替代病理医生。

### 1.3 核心问题

传统病理标注项目通常存在以下问题：

1. 高质量病理标注依赖专家，成本高、周期长。
2. 单个病种、医院或染色方案的数据量有限，模型难以从零训练。
3. 新增样本、类别或中心后，模型需要反复全量训练。
4. 模型推荐结果、人工修订结果和训练数据版本难以追溯。
5. WSI 文件巨大，普通 Web 图像工具无法流畅浏览和标注。
6. 模型更新若缺少验证和审批，可能出现灾难性遗忘或性能倒退。

### 1.4 建设目标

1. 提供流畅的 WSI 多分辨率浏览和多类型标注能力。
2. 通过 AI 预标注、交互式修订和快捷操作降低人工标注成本。
3. 在少量已标注样本下，利用迁移学习、数据增强和主动学习获得可用模型。
4. 将医生修订后的高价值样本纳入增量训练闭环。
5. 对数据集、标注、训练任务、模型、评估和发布过程进行完整版本化。
6. 支持模型灰度使用、效果对比、回滚和审计。

### 1.5 关键术语

| 术语 | 含义 |
| --- | --- |
| WSI | 全切片图像，通常包含多个分辨率层级 |
| ROI | 感兴趣区域 |
| Patch | 从 WSI 某一倍率裁剪得到的局部图像块 |
| 标注集 | 某个版本下，一组样本及其人工复核标注 |
| 预标注 | 模型生成、尚未被人工确认的标注 |
| 金标准 | 经指定角色复核并冻结的高可信标注 |
| 小样本学习 | 利用预训练、数据增强和有限标注样本训练任务模型 |
| 增量学习 | 在保留旧知识的前提下，使用新增数据更新模型 |
| 主动学习 | 优先挑选最值得人工标注的样本 |
| 回放集 | 增量训练时用于保持旧知识的历史代表性样本 |
| 模型门禁 | 模型发布前必须满足的评估规则 |

## 2. 项目边界

### 2.1 首版包含

- 用户、角色、项目和数据权限管理。
- WSI/普通图像导入、校验、元数据解析和缩略图生成。
- WSI 金字塔瓦片浏览。
- 图像级分类、点、矩形、折线、多边形、自由笔刷等标注。
- 标签体系、标注任务和任务分派。
- AI 批量预标注和交互式辅助标注。
- 标注提交、复核、驳回、修改、冻结和冲突处理。
- 主动学习候选队列。
- 小样本训练和批次式增量训练。
- 模型评估、对比、审批、发布、回滚。
- 数据集、标注和模型版本追踪。
- 常见格式导入导出和审计日志。

### 2.2 首版不包含

- 自动生成最终临床诊断或病理报告。
- 无人工审核的在线自动学习。
- 跨医院联邦学习。
- 扫描仪硬件控制。
- 复杂 LIS/HIS/PACS 双向集成。
- 计费、耗材和科室运营管理。
- 多模态大模型自动诊断。

以上能力可作为后续阶段扩展。

## 3. 用户角色

| 角色 | 主要职责 | 关键权限 |
| --- | --- | --- |
| 系统管理员 | 用户、存储、系统参数管理 | 全局配置、用户与角色管理 |
| 项目管理员 | 创建项目、配置标签、分派任务 | 项目内全部数据和流程管理 |
| 病理专家/复核医生 | 制定规范、复核标注、批准金标准 | 复核、冻结、驳回、查看评估 |
| 标注员 | 执行人工标注和修订 | 查看被分派样本、提交标注 |
| 算法工程师 | 配置训练、评估和模型候选版本 | 创建训练任务、查看数据统计 |
| 模型管理员 | 审批、发布和回滚模型 | 模型生产状态变更 |
| 审计员 | 检查数据和模型操作记录 | 只读审计访问 |

同一用户可拥有多个角色；权限应同时受角色、项目成员关系和样本数据范围约束。

## 4. 典型业务流程

### 4.1 新项目冷启动

1. 项目管理员创建项目、任务类型和标签体系。
2. 导入 WSI，系统解析倍率、尺寸、染色和来源等元数据。
3. 病理专家制定标注规范并标记少量种子样本。
4. 算法工程师选择预训练模型，进行小样本微调。
5. 系统在固定验证集上评估，形成首个候选模型。
6. 模型管理员批准后发布为项目辅助标注模型。

### 4.2 AI 辅助标注

1. 用户打开样本，系统加载当前已发布模型的预标注。
2. 用户接受、删除或修改预标注，也可继续手工绘制。
3. 系统记录每个预标注的模型版本和人工修改轨迹。
4. 标注员提交任务，复核医生审核。
5. 通过复核的结果进入可训练数据池。

### 4.3 主动学习与增量训练

1. 系统对未标注池执行推理并计算不确定性、代表性和多样性。
2. 生成建议标注队列，项目管理员可调整优先级。
3. 人工标注并复核候选样本。
4. 当新增金标准达到触发条件时，系统建议发起增量训练。
5. 训练任务使用新增样本、历史回放集和固定验证集。
6. 新模型必须通过新数据效果、旧任务回归和稳定性门禁。
7. 经人工批准后，新版本才可发布；否则保持旧版本。

### 4.4 标注复核

1. 标注员提交后，标注版本进入“待复核”状态。
2. 复核医生查看标注、AI 原结果、人工修改和质量提示。
3. 可选择通过、驳回或直接修订。
4. 通过后的版本成为金标准；冻结后不可直接覆盖，只能创建新版本。
5. 多人标注出现冲突时，由指定专家仲裁。

## 6. 非功能需求

### 6.1 性能

- 在医院内网或等效局域网条件下，常用 WSI 瓦片请求 P95 目标不高于 500 ms；命中缓存时目标不高于 150 ms。
- 首屏应优先显示缩略图和低分辨率层级，逐步加载高分辨率瓦片。
- 单个视野包含数千个矢量对象时，应通过视窗裁剪、空间索引、简化轮廓或分层加载维持可操作性。
- 推理、训练、导出等长任务必须异步执行，不占用 Web 请求进程。
- MVP 目标支持 20 名并发在线标注用户；架构允许水平扩展。

### 6.2 可用性

- 标注草稿周期性保存，浏览器异常关闭后可恢复。
- 后端服务重启不得丢失已确认任务和队列状态。
- 关键服务提供健康检查；任务失败应展示明确原因并允许重试。
- 模型服务不可用时，人工标注功能仍可使用。

### 6.3 安全与隐私

- 全链路使用 TLS；密码采用强哈希算法保存。
- 原始病理数据、导出文件和模型产物按项目与角色授权访问。
- 病例身份信息与图像业务数据逻辑隔离，默认使用脱敏标识。
- 对下载、导出、删除、权限变化和模型发布实施重点审计。
- 支持部署在机构内网或私有云，默认不依赖外部公共服务。
- 正式进入临床环境前，应由使用机构依据适用的数据、网络安全、个人信息保护、医疗器械和伦理要求进行专项合规评审。

### 6.4 可维护性与可扩展性

- 前后端分离，业务 API、图像瓦片服务、任务服务和模型服务边界清晰。
- 图像格式解析、模型推理、训练策略和导出格式采用插件/适配器方式扩展。
- 数据库变更通过迁移脚本管理。
- API、训练配置和内部交换格式均带版本号。
- 关键模块具备自动化测试和可观测指标。

### 6.5 可追溯与可复现

任何生产模型结果必须能够追溯到：

- 模型文件及校验值；
- 代码版本和运行镜像；
- 父模型；
- 训练配置和随机种子；
- 训练、验证和测试数据集版本；
- 标签体系版本；
- 评估报告；
- 审批和发布时间。

## 7. 数据与标注规则

### 7.1 数据层级

```text
项目
└── 病例（去标识化）
    └── 切片
        └── 图像/WSI
            ├── 标注任务
            ├── 标注版本
            ├── AI 推理结果
            └── Patch/特征索引
```

### 7.2 标注来源

标注对象必须区分以下来源：

- `manual`：完全人工创建；
- `model`：模型生成、尚未确认；
- `model_accepted`：模型生成并被人工直接接受；
- `model_corrected`：模型生成后被人工修改；
- `imported`：从外部系统导入。

模型生成的对象只有在人工确认并通过项目复核规则后，才可作为金标准训练数据。

### 7.3 版本规则

- 草稿可被当前编辑者持续更新。
- 每次提交生成不可变的标注版本快照。
- 复核修改通过新版本表达，不覆盖历史版本。
- 标签体系变更必须生成新版本并提供映射关系。
- 数据集版本是样本版本和标注版本的不可变清单。

## 8. 小样本与增量学习策略要求

### 8.1 小样本冷启动

首选策略：

1. 使用通用视觉或病理领域预训练模型提取特征。
2. 先冻结大部分骨干，仅训练任务头或参数高效适配层。
3. 使用颜色、尺度、旋转、翻转和形态相关数据增强。
4. 按病例而非 Patch 划分数据集。
5. 处理类别不平衡，避免被多数类主导。
6. 用交叉验证或重复实验表达小样本结果的不确定性。

### 8.2 增量学习防遗忘

MVP 必须实现：

- 历史代表性样本回放；
- 固定旧任务回归测试集；
- 父模型与候选模型统一评估；
- 可配置的性能退化门禁；
- 版本化发布和一键回滚。

后续可增加：

- 知识蒸馏；
- 正则化约束；
- 原型记忆；
- 类别增量专用分类头；
- 域适配和持续校准。

### 8.3 主动学习优先级

候选评分可综合：

```text
总分 = 不确定性 × w1
     + 代表性 × w2
     + 多样性 × w3
     + 业务优先级 × w4
     - 重复性惩罚 × w5
```

系统必须显示主要入选原因。主动学习结果是建议，不得强制替代病理专家的选样判断。

## 9. MVP 范围与验收标准

### 9.1 MVP 建议范围

- 单机构、私有化部署。
- 分类与区域分割两类任务。
- SVS、TIFF/BigTIFF、NDPI、PNG、JPEG。
- 单人标注、单人复核。
- 基础 AI 预标注和 ROI 交互分割。
- 一种小样本微调模板。
- 基于熵/置信度和特征聚类的主动学习。
- 基于回放数据的批次式增量训练。
- 候选模型评估、人工发布和回滚。
- GeoJSON、COCO、掩膜和 CSV 导出。

### 9.2 功能验收

1. 用户可完成“导入切片—创建任务—标注—提交—复核—导出”完整流程。
2. 用户可对 WSI 进行多倍率浏览，标注坐标在缩放后保持一致。
3. AI 预标注与人工标注来源可区分、可追溯。
4. 已通过标注可生成不可变数据集版本并发起训练。
5. 增量训练包含历史回放集，并输出新旧任务对比报告。
6. 未通过模型门禁的版本无法发布。
7. 已发布模型可回滚到上一版本。
8. 任意生产推理结果可追溯到模型和数据版本。

### 9.3 效果验收建议

效果指标应由具体病种和任务单独确定，平台级建议目标如下：

- AI 辅助后，目标任务的中位标注耗时相对纯人工基线降低至少 30%。
- 已发布模型预标注的人工直接接受率达到项目预设门槛。
- 增量版本在旧任务关键指标上的退化不超过项目门禁值。
- 数据集划分无病例级泄漏。
- 标注、模型和审批链路追溯完整率为 100%。

上述数值属于项目目标，不应被解释为对任意数据和病种的效果承诺。

## 10. 风险与应对

| 风险 | 影响 | 应对 |
| --- | --- | --- |
| 少量样本代表性不足 | 验证结果波动大 | 病例级交叉验证、置信区间、主动学习补样 |
| 染色与扫描仪域偏移 | 新中心性能下降 | 颜色归一化、亚组评估、域增量训练 |
| 伪标签错误累积 | 模型偏差被放大 | 伪标签不得直接进入金标准；必须人工复核 |
| 灾难性遗忘 | 新模型损害旧任务 | 回放集、旧任务测试集、门禁和回滚 |
| 标签规范不稳定 | 历史数据无法复用 | 标签体系版本化和显式映射 |
| WSI 性能不足 | 标注体验差 | 金字塔瓦片、缓存、空间索引、异步任务 |
| 专家意见不一致 | 金标准质量下降 | 双标、仲裁、一致性统计和规范迭代 |
| 数据泄露 | 合规和伦理风险 | 私有部署、脱敏、最小权限、审计与导出控制 |
| 模型指标好但不好用 | 无实际效率收益 | 同时监控接受率、修改量和标注时间 |

## 11. 待产品确认事项

以下事项不阻塞架构设计，但进入开发前应由业务方确认：

1. 首个目标病种、染色类型和标注对象。
2. 首版优先做分类、检测还是分割。
3. 预计 WSI 数量、单文件大小、并发人数和 GPU 资源。
4. 是否需要适配特定扫描仪私有格式。
5. 标注采用单人复核还是双人复核。
6. 是否需要对接现有病例系统，以及可交换的脱敏字段。
7. 模型发布门禁的任务级指标与允许退化范围。
8. 数据保留、备份、导出审批和审计保留周期。
